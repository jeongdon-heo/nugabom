# 기록도우미 (가칭) — 설계 문서

> 초등학교 생활기록부 서술형 항목 작성 지원 도구
> 누가기록 수집 + AI 초안 생성 종합 도구

\---

## 1\. 프로젝트 개요

### 1.1 문제 정의

초등학교 담임교사는 매 학기 말 학생 1인당 3개의 서술형 항목을 작성해야 한다.

|항목|내용|난이도|
|-|-|-|
|**창의적 체험활동상황**|자율·자치, 동아리, 진로활동 영역별 특기사항|중|
|**교과학습발달상황**|교과별 세부능력 및 특기사항|상|
|**행동특성 및 종합의견**|학생의 행동 특성, 성장 과정 종합 서술|상|

25명 학급 기준, 학기 말에 **최소 75개 이상의 서술문**을 작성해야 하며, 이 과정에서 다음 문제가 발생한다:

* **누가기록 부족**: 학기 초·중 관찰 내용이 체계적으로 남아있지 않아 학기 말에 기억에 의존
* **시간 부담**: 1인당 30분\~1시간, 전체 학급 기준 수십 시간 소요
* **유사성 문제**: 피로 누적으로 학생 간 문장이 비슷해지는 현상
* **기재요령 준수**: 학년별, 영역별로 다른 기재 기준을 매번 참조해야 함

### 1.2 해결 방향

**2단계 워크플로우:**

```
\\\[1단계: 평소] 간편 누가기록 → 학생별 관찰 데이터 축적
\\\[2단계: 학기 말] 축적된 데이터 기반 AI 초안 생성 → 교사 검토·수정 → 최종본
```

### 1.3 핵심 원칙

1. **입력은 최소화, 출력은 최대화**: 키워드·짧은 메모만으로 풍부한 서술문 생성
2. **교사가 주인**: AI는 초안만 제공, 최종 판단과 수정은 반드시 교사가 수행
3. **기재요령 준수**: 학년별 기재요령을 내장하여 규정에 맞는 문장 생성
4. **데이터 보안**: 학생 개인정보가 포함되므로 로컬 우선, 외부 전송 최소화

## 2\. 사용자 시나리오

### 시나리오 A: 평소 누가기록 (소요시간 목표: 30초 이내)

> 수업 후 쉬는 시간, 김민준 학생이 수학 시간에 분수의 덧셈 개념을 친구에게 설명해 주는 모습이 인상적이었다.

1. 앱을 열고 **학생 이름 터치** (또는 검색)
2. **영역 선택**: 교과 > 수학
3. **메모 입력**: "분수 덧셈 친구에게 설명, 또래교수 역할"
4. 자동으로 날짜·영역 태깅되어 저장

### 시나리오 B: 교과 성취기준 기반 누가기록 (소요시간 목표: 10초)

> 수학 시간 후, 김민준 학생이 분수의 덧셈 개념을 잘 이해하고 친구에게 설명하는 모습을 기록하고 싶다.

1. 앱 기록 탭 → **학생 터치**: 김민준
2. **교과 선택**: 수학
3. **단원 선택**: 2. 분수의 덧셈과 뺄셈
4. **성취기준 표시**: \[4수01-10] 분모가 같은 분수의 덧셈과 뺄셈을 할 수 있다.
5. **AI 생성 관찰 문구** 클릭: \[개념 이해 우수] \[또래에게 설명]
6. (선택) 추가 메모 입력: "동분모 분수 덧셈 원리를 구체물로 시연"
7. **전송** → 자동으로 교과/수학/성취기준 코드 태깅되어 저장

### 시나리오 C: 학기 말 AI 초안 생성

> 학기 말, 김민준 학생의 교과학습발달상황 세부능력 및 특기사항을 작성해야 한다.

1. **초안 생성** 메뉴 진입
2. 학생 선택 → 해당 학기 누가기록 목록 확인
3. **항목 선택**: 교과학습발달상황
4. AI가 누가기록 + 학년 기재요령을 기반으로 초안 생성
5. 교사가 초안을 **검토·수정** → 저장
6. 최종본 일괄 **복사/내보내기** → 나이스에 붙여넣기

### 시나리오 C: 빈 기록 보완

> 학기 중반, 앱이 알림: "이서연 학생의 창체 영역 기록이 아직 없습니다."

1. 대시보드에서 **기록 현황 매트릭스** 확인
2. 기록이 부족한 학생·영역 한눈에 파악
3. 해당 학생에 대한 관찰 메모 추가 입력

## 3\. 정보 구조 (Information Architecture)

```
기록도우미
├── 🏠 홈 (대시보드)
│   ├── 학급 기록 현황 매트릭스 (학생 × 영역)
│   ├── 최근 기록 타임라인
│   └── 기록 부족 알림
│
├── ✏️ 누가기록
│   ├── 빠른 입력 (학생 선택 → 영역 → 메모)
│   ├── 학생별 기록 보기
│   └── 영역별 기록 보기
│
├── 🤖 AI 초안 생성
│   ├── 학생 선택
│   ├── 항목 선택 (창체 / 교과 / 행동특성)
│   ├── 초안 생성 및 편집
│   └── 일괄 생성 모드
│
├── 📋 최종본 관리
│   ├── 학생별 완성본 보기
│   ├── 일괄 내보내기 (텍스트/엑셀)
│   └── 나이스 붙여넣기용 복사
│
└── ⚙️ 설정
    ├── 학급 정보 (학년, 학생 명단)
    ├── 교과 목록 (학년별 자동 설정)
    └── 기재요령 버전 선택
```

## 4\. 데이터 모델

### 4.1 핵심 엔티티

```
┌─────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  Classroom   │────<│     Student      │────<│  Observation    │
│  (학급)       │     │     (학생)        │     │  (누가기록)      │
├─────────────┤     ├──────────────────┤     ├─────────────────┤
│ id           │     │ id               │     │ id              │
│ year         │     │ classroomId      │     │ studentId       │
│ semester     │     │ number (번호)     │     │ date            │
│ grade (학년)  │     │ name             │     │ domain          │
│ className    │     │ gender           │     │ category        │
└─────────────┘     └──────────────────┘     │ memo            │
                                              │ tags\\\[]          │
                                              └─────────────────┘

┌─────────────────┐
│     Draft       │
│   (AI 초안)      │
├─────────────────┤
│ id              │
│ studentId       │
│ recordType      │  ← "창체" | "교과" | "행동특성"
│ subject?        │  ← 교과인 경우 과목명
│ generatedText   │  ← AI 생성 원문
│ editedText      │  ← 교사 수정본
│ status          │  ← "draft" | "reviewed" | "final"
│ createdAt       │
│ updatedAt       │
└─────────────────┘
```

### 4.2 영역(Domain) 분류 체계

```
domain (대분류)
├── "창체"
│   ├── category: "자율자치"
│   ├── category: "동아리"
│   └── category: "진로"
│
├── "교과"
│   ├── category: "국어"
│   ├── category: "수학"
│   ├── category: "사회" / "바른생활" (1-2학년)
│   ├── category: "과학" / "슬기로운생활" (1-2학년)
│   ├── category: "영어" (3학년\\\~)
│   ├── category: "도덕" (3학년\\\~)
│   ├── category: "체육"
│   ├── category: "음악"
│   ├── category: "미술"
│   └── category: "실과" (5-6학년)
│
└── "행동특성"
    ├── category: "학습태도"
    ├── category: "교우관계"
    ├── category: "생활습관"
    ├── category: "리더십"
    ├── category: "정서"
    └── category: "기타"
```

> \\\*\\\*학년별 교과 자동 매핑\\\*\\\*: 학급 설정에서 학년을 지정하면 해당 학년에 맞는 교과 목록이 자동으로 표시된다. (예: 1-2학년 → 통합교과, 3학년\\\~ → 영어 추가, 5학년\\\~ → 실과 추가)

### 4.3 저장 방식 후보

|방식|장점|단점|적합도|
|-|-|-|-|
|**LocalStorage / IndexedDB**|설치 불필요, 즉시 사용|용량 제한, 기기 종속|★★★ (프로토타입)|
|**Supabase**|실시간 동기화, 인증 내장|외부 서버에 학생 데이터 저장|★★☆|
|**SQLite (로컬 파일)**|보안 우수, 대용량|웹앱에서 직접 사용 어려움|★★☆|
|**JSON 파일 내보내기/가져오기**|가장 단순, 백업 용이|실시간 동기 없음|★★★ (보조)|

**권장안**: IndexedDB를 기본 저장소로 사용하되, JSON 내보내기/가져오기 기능을 통해 백업과 기기 간 이동을 지원한다. 민감 데이터 보호를 위해 외부 서버 저장은 최소화한다.

## 5\. AI 초안 생성 설계

### 5.1 생성 전략

AI에게 전달하는 정보:

```
\\\[컨텍스트]
- 학년: 4학년
- 학기: 1학기
- 항목: 교과학습발달상황 > 수학

\\\[기재요령 가이드]
- 성취기준과 성취수준에 근거하여 학생 개인의 성취과정과 성취특성이 
  명료히 드러나도록 기재
- 수업에서 이루어진 활동의 단순 나열이나 성취기준에 명시된 지식의 
  단순 서술은 지양
- 학생의 성장과 변화를 중심으로 서술

\\\[학생 누가기록]
- 3/15: 분수 덧셈 친구에게 설명, 또래교수 역할
- 4/22: 각도기 사용에 어려움 → 연습 후 정확하게 측정 성공
- 5/10: 막대그래프 프로젝트에서 주제 선정 및 자료 수집 주도
- 6/03: 규칙찾기 단원 패턴 일반화 잘함

\\\[요청]
위 누가기록을 종합하여 교과학습발달상황 세부능력 및 특기사항 초안을 
작성해 주세요. 기재요령을 준수하고, 학생의 성장 과정이 드러나도록 
해 주세요.
```

### 5.2 항목별 프롬프트 전략

#### 창의적 체험활동상황

```
- 영역별(자율·자치, 동아리, 진로) 특기사항을 구분하여 작성
- 활동 내용과 함께 학생의 태도, 변화, 성장을 서술
- 학년 특성에 맞는 수준의 표현 사용
- 기재 분량: 영역별 1\\\~3문장
```

#### 교과학습발달상황

```
- 교과별로 구분하여 작성
- 성취기준·성취수준 기반 서술 (단순 나열 금지)
- 학생의 강점과 성장 과정 중심
- 구체적 활동 사례 포함
- 기재 분량: 교과별 2\\\~4문장
```

#### 행동특성 및 종합의견

```
- 학생의 인성, 사회성, 학습 태도 등을 종합적으로 서술
- 긍정적 측면 중심으로 작성하되 성장 가능성도 포함
- 담임교사의 관찰 소견 형식
- 기재 분량: 5\\\~10문장
```

### 5.3 AI 모델 선택

|모델|비용|품질|속도|적합도|
|-|-|-|-|-|
|**Gemini 2.0 Flash**|무료/저비용|양호|빠름|★★★|
|**Claude Sonnet**|중간|우수|보통|★★☆|
|**GPT-4o mini**|저비용|양호|빠름|★★☆|

**권장안**: 학급 전체 일괄 생성 시 비용을 고려하면 Gemini 2.0 Flash가 현실적. Claude Sonnet은 품질이 중요한 경우 선택적 사용.

### 5.4 품질 관리 장치

1. **유사도 검사**: 같은 학급 내 학생 간 생성 문장 유사도를 자동 체크하여 경고
2. **기재요령 체크리스트**: 생성된 문장이 기재요령 금지사항(학교명 기재, 과장 표현 등)을 위반하지 않는지 자동 검증
3. **누가기록 매핑**: 생성된 문장의 각 부분이 어떤 누가기록에서 파생되었는지 표시
4. **분량 가이드**: 항목별 권장 글자 수 범위 표시

  교과: \\\["국어", "수학", "사회", "과학", "영어", "도덕", 
         "체육", "음악", "미술", "실과", "바른생활", "슬기로운생활"],
  창체: \\\["자율", "자치", "동아리", "진로"],

};

function parseVoiceInput(text) {
  // "수학 분수 덧셈 친구에게 설명함"
  // → { domain: "교과", category: "수학", memo: "분수 덧셈 친구에게 설명함" }
  
  for (const \\\[domain, keywords] of Object.entries(VOICE\\\_KEYWORDS)) {
    for (const keyword of keywords) {
      if (text.startsWith(keyword) || text.includes(keyword + " ")) {
        const memo = text.replace(keyword, "").trim().replace(/^\\\[,\\\\s]+/, "");
        return { domain, category: keyword, memo };
      }
    }
  }
  return { domain: null, category: null, memo: text };
}

## 10\. 보안 및 개인정보 고려사항

### 10.1 데이터 분류

|데이터|민감도|처리 방침|
|-|-|-|
|학생 이름·번호|높음|로컬 저장, AI 전송 시 번호만 사용 가능|
|누가기록 내용|중간|AI 전송 필요, 최소 정보만 전달|
|AI 생성 초안|낮음|로컬 저장|

### 10.2 AI 전송 시 익명화 옵션

```
원본: "김민준이 수학 시간에 분수 덧셈을 친구에게 설명함"
익명화: "해당 학생이 수학 시간에 분수 덧셈을 친구에게 설명함"
```

AI에게 전송 시 학생 이름을 "해당 학생"으로 치환하고, 응답 수신 후 다시 이름으로 복원하는 방식으로 개인정보 노출을 최소화할 수 있다.

### 10.3 데이터 삭제

* 학기 종료 후 데이터 일괄 삭제 기능 제공
* 학급 변경 시 이전 학급 데이터 아카이브 또는 삭제 선택
